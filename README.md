# Costco Wholesale India — Global Capability Center

A static, single-page Next.js site for Costco Wholesale India Private Limited's GCC. Built to deploy on GitHub Pages.

## Stack

- Next.js 14 App Router with `output: 'export'`
- Tailwind CSS, Framer Motion, lucide-react
- Client-only rule-based AI chatbot (Costa)
- WebAuthn passkey demo at `/portal/` (frontend-only, hidden from nav)
- DPDP / GDPR-aware cookie consent with category preferences
- WCAG 2.2 AA-targeted: skip link, focus styles, reduced motion, semantic landmarks

## Run locally

```bash
npm install
npm run dev   # http://localhost:3000
npm run build # produces ./out for static deploy
```

## Deploy to GitHub Pages

1. Push this repo to GitHub.
2. In **Settings → Pages**, set source to **GitHub Actions**.
3. The workflow at `.github/workflows/deploy.yml` runs on push to `main`:
   - It sets `NEXT_PUBLIC_BASE_PATH` automatically based on the repo name.
   - For `username.github.io` → root path. For any other repo → `/<repo>/`.

## Customize

- **Logo** — `components/Logo.tsx` (and `public/favicon.svg` for the browser tab).
- **Copy** — sections in `components/sections/*`.
- **Chatbot knowledge** — `lib/chatbot-knowledge.ts` (intents and replies).
- **Careers feed** — Powered by `public/openings.json`, which is generated by `npm run sync:openings`. The script ([scripts/sync-openings.mjs](scripts/sync-openings.mjs)) calls Talent500's public API (`https://prod-warmachine.talent500.co/api/v3/jobs/search/?company_slug=costco`) with browser-style `Origin`/`Referer` headers — no auth required — and writes a structured list of all live Costco roles. A daily GitHub Actions workflow ([`.github/workflows/sync-openings.yml`](.github/workflows/sync-openings.yml)) runs the sync at 03:30 UTC and commits the file if it changed. To trigger manually: Actions → "Sync Talent500 openings" → "Run workflow". The Careers section ([components/sections/Careers.tsx](components/sections/Careers.tsx)) reads the JSON at runtime and renders dynamic category filters with skill chips. No redeploy is needed when the JSON updates.
- **Locations** — `components/sections/Locations.tsx`. Currently set to **Bengaluru** and **Hyderabad** — change there if the user-intended "Capital" was Delhi NCR or another city.
- **Legal pages** — `app/legal/*`. Templates only; have Privacy/Legal review before publishing.
- **Hidden portal** — `app/portal/page.tsx`. Reachable at `/portal/`. To fully hide from search, `robots.txt` already disallows.

## Compliance scaffolding

- Cookie consent: explicit categories (Necessary always-on, Preferences, Analytics, Marketing).
- A floating shield icon re-opens preferences any time.
- Privacy notice references DPDP, GDPR, retention, rights.
- Accessibility statement and Whistleblower policy included.
- No third-party scripts loaded by default.

## What's intentionally not here

- No analytics, ad pixels, or social SDKs are loaded — wire them only behind `analytics`/`marketing` consent.
- No backend. The chatbot is rule-based; the contact form opens the user's email client.
- Passkey "login" is a UX demo — pair with an IdP (e.g. Okta, Azure AD, Auth0) before production.
